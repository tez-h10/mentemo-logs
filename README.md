# mentemo logs

会社では**広告運用**と**開発**が主軸です。このリポジトリは、そのログ・設定・要件を**案件・事業ごと**に整理するためのものです。

## 構成

| フォルダ | 内容 |
|----------|------|
| **[Google広告運用](./Google広告運用/)** | 会社共通。SSID/LTIDマスタ・GL→スプシの雛形・config（ピッカーズとSSID案件は**並列**で、ここを参照） |
| **[ピッカーズ](./ピッカーズ/)** | 広告運用案件。施策ログ・議事録（マーケ・運用の拠点）。**例外で SSID/LTID は持たない** |
| **[エザキ](./エザキ/)** | 広告運用案件。カーリース・Keeper（コーティング）の2本柱。概要・logs |
| **[清亀石油](./清亀石油/)** | 広告運用案件。朝霞・車検LP。会社のSSID/LTID・GL→スプシの枠で運用。logs |
| **[ジュエリノ](./ジュエリノ/)** | 自社事業。全体像（共通）・マーケ・開発・logs |
- マーケは施策・KPI・全体像を主に参照・更新。開発向けの要件はジュエリノ/開発にまとめます。

## よく使う検索

- **全案件のタスク一覧（こっちがボール）**: [タスク一覧.md](./タスク一覧.md)
- **会社名 → SSID / SSID → 会社名**: [Google広告運用/SSID一覧.md](./Google広告運用/SSID一覧.md) を開いて検索（Ctrl+F / Cmd+F）
- **LP URL（昭和・清亀など）**: [Google広告運用/LP移行方針.md](./Google広告運用/LP移行方針.md)

## 運用

- 案件ごとにフォルダを分け、中で施策ログ（data / logs）や広告設定・開発要件を管理
- **ログは各案件フォルダ内の logs で管理** … 必要な情報のみ残す。全ログは貯めず要点だけ（容量を抑える）。メール文全文・毎回の生データは保存しない
- **ルートに logs/（全ログ一覧）は作らない** … 案件別の logs のみ運用する
- **案件（会社）ごとにフォルダを作り管理** … エザキ・ジュエリノ・清亀石油も各フォルダ内の logs で完結。清亀石油は会社の広告運用の枠（SSID/LTID・GL→スプシ）で運用し、ログは 清亀石油/logs に記録する
- **雛形は保存** … スクリプト・テンプレ・設定の型はリポジトリに残す
- 日付は `YYYY-MM-DD` 形式
