# エージェント・業務活用の方向性

このリポジトリ（mentemo logs）を**日々の業務ツール**として使い、会話とログから施策の可視化・メール返信・マーケPDCA・開発要件までつなげていくための方針です。

---

## 目指していること

1. **メール文の生成** … 日々の業務でメール文作成をエージェントに依頼する。
2. **会話からログを抽出** … 全ての会話からログを抜き出し、**現在進行中の施策**を洗い出せるようにする。
3. **先方メールへの返信文の自走** … 過去のログ・議事録・送付履歴を参照し、精度高く返信文を書けるようにする。こちらから自発的に議事録などを渡して情報を貯める。
4. **マーケ業務** … ワイヤーフレーム検討など。過去の施策・検証した訴求軸・得られた結果からPDCAを回し、次の作成（LP・クリエイティブ・ワイヤーなど）に反映する。
5. **開発の要件定義** … 機能要件・仕様・受け入れ条件などを整理し、開発向けの要件定義もエージェントと一緒にまとめていく。

---

## いまの土台（リポジトリでやっていること）

- **案件別の logs** … 各案件フォルダ内の `logs/` にサマリのみ残す。会話で得た要点・送付済み・施策メモをここに追記していく。
- **議事録の格納** … ピッカーズは `施策ログ/data/議事録/` に日付別で保存。他案件も必要に応じて data や logs に「何をいつ渡したか」を残す。
- **雛形・マスタ** … Google広告運用のconfig、SSID一覧、案件概要などはリポジトリに置き、エージェントが参照できるようにする。
- **開発要件の置き場** … ジュエリノは `ジュエリノ/開発/` と `全体像/` に要件・仕様をまとめる。他案件でも開発向けはフォルダを分けて管理。

## 今後そろえていきたいこと

| 目的 | やること |
|------|----------|
| 進行中施策の洗い出し | 会話ログ＋各案件の logs を参照して「いま何が動いているか」を一覧・要約する。必要ならログに「進行中」や種別を付与。 |
| 返信文の精度向上 | 議事録・過去のメール要約・送付履歴を渡す／logs に残す。エージェントは案件のログ＋渡された議事録を前提に返信文を書く。 |
| マーケ・PDCA | 施策結果・訴求軸・検証内容を data や logs に構造化して残す。ワイヤー・LP依頼時に「過去の結果から次の案」を出せるようにする。 |
| 開発の要件定義 | 会話や議事録から「やりたいこと」を抽出し、機能要件・仕様・受け入れ条件として整理。ジュエリノ/開発 や案件別の開発フォルダに追記・更新する。 |

---

## 運用上の前提

- **こちらから自発的に議事録などを渡す** … エージェントが参照できる情報が増えるほど、ログ抽出・返信・要件定義の精度が上がる。
- **ログは「必要な情報のみ」** … 全文を貯めず、判断・参照に必要な要点だけ残す。メール生成・返信・要件のための要約・キーワード・日付・宛先はログに含める。
- **案件ごとにフォルダを分ける** … どの会話・ログがどの案件か分かるようにする。返信・施策洗い出し・要件定義は案件単位で参照する。
